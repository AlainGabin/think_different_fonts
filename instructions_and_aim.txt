I want you to build a lightweight, native Windows desktop utility application (written in C# using WPF, or native C++ with a clean UI) named "MacFontRenderer for Windows". 

The primary goal of this application is to completely refactor the Windows font-rendering engine system-wide to mimic macOS grayscale antialiasing and typography thickness on 4K/High-DPI monitors at 100% scale without zooming.

The application must contain the following features and architectural layout:

1. ADMIN PRIVILEGES REQUREMENT:
The application must check for Administrator privileges upon execution. If not run as admin, prompt the user to restart the application with elevated rights, as it modifies HKEY_LOCAL_MACHINE registry keys and system directories.

2. FONT MANAGEMENT INTEGRATION:
- Build a routine that checks if the "SF Pro Text" or "Inter" font family is installed on the system.
- If missing, the app should download a bundled zip containing the clean TrueType/OpenType files for Regular and Bold variants, extract them, and systematically invoke the `AddFontResourceEx` API and update `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts` to install them system-wide for "All Users".

3. THE AUTOMATED TWEAKS PIPELINE (The "Apply Mac Style" Button):
When the user clicks the "Optimize System Fonts" button, execute these exact operations sequentially:
  a. Registry Injection (GDI Engine): Update `HKCU\Control Panel\Desktop` -> Set FontSmoothing="2", FontSmoothingType=2, and FontSmoothingGamma=1000 (Decimal). Provide a slider in the UI to allow adjusting Gamma down to 336 for an extra-heavy ink look.
  b. Environment Injection (DirectWrite Engine): Update `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment` -> Set GDIP_FONT_SMOOTHING="2" and DWRITE_FONT_SMOOTHING="2". Broadcast the change using `SendMessageTimeout(HWND_BROADCAST, WM_SETTINGCHANGE, ...)` so apps immediately catch the environment update.
  c. Early-Boot Substitution: Update `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Fonts` to map all variants of "Segoe UI" directly to the chosen installed font filename (e.g., "SF-Pro-Text-Regular.otf").
  d. AppInit FontMod Hook Deployment: Create a hidden application data folder. Drop a programmatically compiled `FontMod.dll` engine inside. Automatically write a perfectly indented `FontMod.yaml` file targeting Segoe UI, Arial, Tahoma, MS Shell Dlg, and Calibri, routing them all to the replacement target font. Update both the 64-bit and 32-bit (Wow6432Node) `AppInit_DLLs` paths to point to this directory, and flip `LoadAppInit_DLLs` to 1.

4. SAFETY ROLLBACK ("Restore Windows Defaults" Button):
Create an absolute fail-safe rollback method. When triggered, it must gracefully clear the AppInit variables, delete the environment variables, restore FontSmoothingGamma back to 4096 (Hex 1000), and remap the early-boot fonts back to default strings ("segoeui.ttf", "segoeuib.ttf", etc.) to completely prevent bricking the user's operating system UI layout.

5. USER INTERFACE REQUIREMENTS:
The interface must be hyper-minimalistic, distraction-free, and clean. Provide a status indicator checking if the optimization is active, a slider for font thickness/weight (Gamma), an "Apply Changes" button, a "Restore Defaults" button, and an explicit warning indicating that a system restart is required for file-level kernel hooks to take effect.

Provide the complete, clean, production-ready source code structure with error-handling blocks wrapped around every Registry modification to ensure system stability.
